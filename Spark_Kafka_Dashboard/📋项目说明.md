# 📋 项目具体说明

> **项目名称**：基于Kafka+Spark+HDFS的电商实时大数据分析平台  
> **课程**：大数据技术与应用  
> **技术栈**：Kafka + PySpark(Core/SQL/Streaming/MLlib) + HDFS + Flask + ECharts + SocketIO  

---

## 一、项目做了什么

用一句话概括：**模拟电商交易数据，通过大数据组件实时处理和分析，最终以Web可视化大屏展示分析结果。**

具体来说：
1. 写了一个数据生成器，每2秒自动生成50条模拟的电商交易记录
2. 这些数据经过 PySpark 实时聚合计算（按城市、性别、支付方式、消费类型等维度）
3. 计算结果通过 SocketIO 实时推送到前端
4. 前端用 ECharts 渲染成7个交互式可视化页面
5. 同时数据会存入 HDFS 分层存储，PySpark SQL 做离线多维分析

### 大数据组件（5个）

| # | 组件 | 在本项目中的具体用途 |

| 1 | **Kafka** | 消息队列，实时采集和传输交易数据（生产→Topic→消费） |

## 五、技术亮点（加分项）

### 1. 三种运行模式设计
系统设计了本地→PySpark→全组件的三级运行模式，可根据环境自动降级，体现了**工程化思维**。

### 2. HDFS接口 + 本地存储双模
`hdfs_storage.py` 封装了统一的存储接口，在Windows使用本地文件系统模拟HDFS，切换到Linux集群时只需改一个配置项。

### 3. 实时+离线双处理
- Spark Streaming 负责2秒级实时聚合（给ECharts）
- Spark SQL 负责T+0离线分析（生成报表）
- 两者互补，覆盖不同时效性需求

### 4. 暗色科技风大屏
BI仪表盘采用深色主题设计，适合数据中心大屏展示场景，视觉效果专业。

### 5. PySpark替代Scala
原参考项目使用Scala（代码已损坏），本项目用PySpark重写全部处理逻辑，Python生态更易维护和修改。

## 六、数据库/存储说明

虽然本项目使用文件系统模拟HDFS（Windows环境限制），但设计了完整的数据分层：

```
HDFS根目录/
├── raw_data/      原始交易JSON（按时间戳命名）
│   raw_20260601_111949.json
│   raw_20260601_111951.json
│   ...
├── processed/      Spark处理中间结果
├── warehouse/      数据仓库分区存储（模拟Hive表结构）
├── analytics/      分析报表（KPI + 多维聚合）
│   analytics_20260601_112002.json
└── checkpoint/     Spark Streaming检查点（故障恢复）
```

切换到真实HDFS时，只需将本地路径替换为 `hdfs://namenode:8020/`。

## 八、答辩可能被问到的问题

### Q1: HDFS在你的Windows上怎么跑的？
**答**：我实现了一个HDFS兼容接口层，Windows开发环境下使用本地文件系统模拟HDFS的分层存储结构，代码接口与真实HDFS完全一致。部署到Linux集群时，只需将 `base_path` 改为 `hdfs://namenode:8020/` 即可切换。

### Q2: Kafka在哪里？我没看到它运行。
**答**：系统设计了fallback机制——当检测不到Kafka时，数据通过内存管道直接流转。Kafka的Docker部署配置已写好（`docker/docker-compose.yml`），在有Docker的Linux环境下一键启动即可接入真实的Kafka。

### Q3: Spark和Hive的关系？
**答**：本项目使用Spark SQL来实现Hive的数据仓库功能。Spark SQL支持标准的Hive SQL语法，可以直接查询结构化数据，在本项目中用于OLAP多维分析。这也是业界当前的主流趋势——Spark SQL正在逐步替代传统Hive。

### Q4: 你做了什么创新？
**答**：1) 三级降级运行模式，适配不同部署环境；2) HDFS接口抽象层，开发-生产无缝切换；3) BI仪表盘暗色大屏设计；4) 实时+离线双处理引擎协同。

### Q5: 数据是假的，实际怎么对接真实数据？
**答**：只需修改 `data_generator.py` 替换为真实的API调用或数据库CDC连接器（如Maxwell、Canal）。其他所有组件（Kafka、Spark、HDFS、可视化）无需修改，因为它们处理的是标准JSON格式。
