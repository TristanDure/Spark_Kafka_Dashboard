## 1. 系统概述

### 1.1 项目背景

随着电子商务的迅猛发展，电商平台每天产生海量的用户行为数据，包括浏览、点击、下单、支付等操作。如何高效地采集、存储、处理和分析这些数据，并从中提取有价值的商业洞察，是当前大数据技术应用的核心课题。

### 1.2 项目目标

本项目设计并实现一个**基于主流大数据组件的电商实时分析平台**，具备以下能力：

- **实时数据采集**：模拟电商用户行为数据生成，通过消息队列实时传输
- **离线+实时数据处理**：使用Spark进行批处理和流式分析
- **分布式存储**：基于HDFS的可靠数据存储
- **交互式可视化**：基于Flask + ECharts + SocketIO的Web实时大屏展示

### 1.3 系统功能概述

| 功能模块 | 说明 |
|----------|------|
| 数据生成引擎 | 模拟电商用户交易行为，生成包含用户画像、城市、消费类型、支付方式等维度的数据 |
| 消息队列传输 | 基于Kafka的实时数据流转 |
| Spark离线分析 | 用户画像统计、热门商品分析、转化率计算、Spark SQL多维分析 |
| Spark Streaming实时分析 | 实时聚合城市/性别/支付方式/消费类型等维度指标 |
| 分布式存储 | 基于HDFS的数据分层存储（原始数据层 / 处理结果层 / 分析结果层） |
| BI可视化大屏 | KPI指标卡、实时折线图、饼图、柱状图、地域分布图等多维度展示 |

### 1.4 大数据组件覆盖

本系统综合使用了 **5个** 大数据生态核心组件：

| 组件 | 用途 | 版本 |
|------|------|------|
| **Apache Kafka** | 消息队列，实时数据流的采集与传输 | 3.6.0 |
| **Apache Spark (Core/SQL/Streaming/MLlib)** | 批处理、实时流处理、SQL分析、机器学习 | 4.1.2 (PySpark) |
| **Apache Hadoop HDFS** | 分布式文件系统，存储原始数据和计算结果 | 3.3.6 (兼容) |
| **Apache Hive (Spark SQL)** | 数据仓库查询，多维OLAP分析 | Spark SQL内置 |
| **ZooKeeper** | 分布式协调服务 | 7.5.0 (Kafka KRaft模式内置) |

### 2.2 各功能模块设计

#### 2.2.1 数据生成模块

**功能**：模拟真实的电商用户交易行为，生成结构化的用户行为数据。

**数据字段**：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| user_id | String | 用户唯一标识 |
| name | String | 用户名 |
| gender | String | 性别（男/女）|
| age_group | String | 年龄段（18-25/26-35/36-45/46-55/55+）|
| city | String | 所在城市（北京/上海/广州/深圳/杭州/成都/武汉/南京）|
| device | String | 使用设备（iOS/Android/PC/H5）|
| goods_type | String | 消费类型（交通/餐饮/娱乐/教育/住房/其他）|
| amount | Float | 交易金额（1-9999元）|
| payment | String | 支付方式（银联/支付宝/微信/现金）|
| action | String | 用户行为（view/click/add_cart/purchase/pay）|
| timestamp | String | 时间戳 |

**设计要点**：
- 每批次生成50条记录，模拟高并发场景
- 使用随机算法生成多样化数据，覆盖完整维度组合
- 数据以JSON格式序列化，便于Kafka传输和Spark解析

#### 2.2.2 Kafka消息队列模块

**功能**：作为数据流转中枢，解耦数据生产与消费。

**Topic设计**：
- `ecommerce_raw`：原始交易数据
- `ecommerce_processed`：Spark处理后的聚合结果

**设计要点**：
- 采用发布-订阅模式，支持多消费者
- 生产者每2秒推送一批数据
- 消费者实时拉取并转发至SocketIO

#### 2.2.3 Spark数据处理模块

**功能**：核心计算引擎，承担离线批处理和实时流计算。

**四大子模块**：

| 子模块 | 技术 | 功能 |
|--------|------|------|
| Spark Core | RDD操作 | Session聚合、数据转换、多维过滤 |
| Spark SQL | DataFrame + SQL | 即席查询、OLAP多维分析 |
| Spark Streaming | DStream | 实时消费Kafka数据、滑动窗口聚合 |
| MLlib | K-Means | 用户分群、RFM模型构建 |

#### 2.2.4 HDFS存储模块

**功能**：分布式文件存储，保障数据可靠性和可扩展性。

**数据分层**：

```
/hdfs_root/
├── raw_data/       # 原始交易数据（JSON格式）
├── processed/      # Spark处理后的中间结果
├── warehouse/      # 数据仓库（分区存储，模拟Hive表）
├── analytics/      # 分析报表（聚合结果、KPI指标）
└── checkpoint/     # Spark Streaming检查点
```

#### 2.2.5 Web可视化模块

**功能**：基于Flask + ECharts + SocketIO的交互式BI大屏。

**页面设计**：

#### 3.3.2 实时流处理

```python
def process_stream_batch(self, records):
    """处理一批实时数据，按维度聚合"""
    city_amount = defaultdict(float)
    gender_amount = defaultdict(float)
    payment_amount = defaultdict(float)
    goods_amount = defaultdict(float)

    for r in records:
        amount = float(r.get("amount", 0))
        city_amount[r.get("city")] += amount
        gender_amount[r.get("gender")] += amount
        payment_amount[r.get("payment")] += amount
        goods_amount[r.get("goods_type")] += amount

    # 返回ECharts所需的16维度数组
    result = []
    result.extend([city_amount.get(c, 0) for c in CITIES])
    result.extend([gender_amount.get(g, 0) for g in GENDERS])
    result.extend([payment_amount.get(p, 0) for p in PAYMENTS])
    result.extend([goods_amount.get(g, 0) for g in GOODS_TYPES])
    return result
```

### 3.4 HDFS存储模块实现

**数据分层存储与读取**：

```python
class HDFSStorage:
    def write_raw(self, data):
        """写入原始数据层 → /hdfs_root/raw_data/"""
        path = f"{self.base_path}/raw_data/raw_{timestamp}.json"
        with open(path, 'w') as f:
            json.dump(data, f, ensure_ascii=False)
        return path

    def write_analytics(self, metrics):
        """写入分析结果层 → /hdfs_root/analytics/"""
        path = f"{self.base_path}/analytics/analytics_{timestamp}.json"
        metrics["_timestamp"] = time.strftime('%Y-%m-%d %H:%M:%S')
        with open(path, 'w') as f:
            json.dump(metrics, f, ensure_ascii=False, indent=2)
        return path
```

## 4. 结论

### 4.1 解决的问题

1. **大数据组件集成**：成功集成了Kafka、Spark(Core/SQL/Streaming)、HDFS、Hive(Spark SQL)共5个大数据组件，覆盖了数据采集、传输、处理、存储、分析、可视化的完整链路。

2. **实时+离线双模式**：通过Spark Streaming实现秒级实时聚合，通过Spark SQL实现T+0离线多维分析，两者结合满足不同时效性需求。

3. **数据可视化**：基于Flask+SocketIO+ECharts构建了7个页面的交互式BI大屏，支持KPI卡片、折线图、饼图、柱状图、地图散点图等多种可视化形式。

4. **可部署性**：系统支持多种运行模式（本地模式/Kafka模式/PySpark模式），对环境依赖最小化，可在Windows/Linux/macOS上运行。

### 4.2 创新点

1. **组件降级运行设计**：当PySpark或Kafka不可用时，系统自动切换到本地处理模式，保证核心可视化功能始终可用。
2. **HDFS本地模拟层**：在开发/Windows环境下使用本地文件系统模拟HDFS接口，代码无需修改即可切换到真实HDFS集群。
3. **暗色主题大屏**：采用深色科技风主题设计，适合数据中心大屏展示场景。

### 4.3 展望

1. **接入真实数据源**：对接真实电商API或数据库CDC，替换模拟数据生成器。
2. **引入Flink**：对于超低延迟场景，可替换Spark Streaming为Flink。
3. **部署到K8s**：将各组件容器化并部署到Kubernetes集群，实现弹性伸缩。
4. **增强ML能力**：引入Spark MLlib的更多算法（推荐系统、异常检测、时序预测）。
